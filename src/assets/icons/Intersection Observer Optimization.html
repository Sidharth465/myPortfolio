Your AnimatedSection component looks well-structured for handling visibility-based animations using the Intersection Observer API. Here's a quick breakdown and a few suggestions:

What You Did Well:

  • Intersection Observer: You're correctly using it to trigger animation when the section comes into view.

  • Unobserve After Trigger: Efficiently unobserving after the first visibility is smart for performance.

  • Reusability: The component is reusable with children and optional id.

Suggestions for Improvement:

1. Observer Cleanup Edge Case

If the section was never visible, observer.unobserve() could fail silently. Consider storing the observer instance in a ref:

tsx
const observerRef = useRef<IntersectionObserver | null>(null);

useEffect(() => {
  observerRef.current = new IntersectionObserver(
    (entries) => {
      if (entries[0].isIntersecting) {
        setIsVisible(true);
        if (sectionRef.current) {
          observerRef.current?.unobserve(sectionRef.current);
        }
      } else {
        setIsVisible(false);
      }
    },
    { threshold: 0.3 }
  );

  if (sectionRef.current) {
    observerRef.current.observe(sectionRef.current);
  }

  return () => {
    if (sectionRef.current) observerRef.current?.unobserve(sectionRef.current);
    observerRef.current?.disconnect();
  };
}, []);

2. CSS Class Handling

Make sure your fade-in and fade-out classes are defined properly in your CSS with transitions, for example:

css
.animated-section {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 0.6s ease-out, transform 0.6s ease-out;
}

.fade-in {
  opacity: 1;
  transform: translateY(0);
}

.fade-out {
  opacity: 0;
  transform: translateY(20px);
}

3. Optional Improvement: Animate Once

If you want the animation to only trigger once and stay visible:

tsx
if (entries[0].isIntersecting) {
  setIsVisible(true);
  observer.unobserve(entries[0].target);
}

What's Going Wrong?

You're unobserving the element as soon as it becomes visible which means the observer is no longer watching it. So it never detects when the element leaves the viewport again, hence setIsVisible(false) doesn't run.

Fix: Only Unobserve if You Don't Want to Toggle

If you want fade-in and fade-out to toggle based on scroll visibility (enter and leave), don't unobserve:

tsx
useEffect(() => {
  const observer = new IntersectionObserver(
    (entries) => {
      if (entries[0].isIntersecting) {
        setIsVisible(true);
      } else {
        setIsVisible(false);
      }
    },
    { threshold: 0.3 }
  );

  if (sectionRef.current) {
    observer.observe(sectionRef.current);

  return () => {
    if (sectionRef.current) observer.unobserve(sectionRef.current);

}, []);
